# org-zk

A lightweight [Zettelkasten](https://en.wikipedia.org/wiki/Zettelkasten) for Emacs, built on plain org-mode and Doom's bundled completion stack (Vertico + Consult + ripgrep).

## Conventions

- Each note ("zettel") is a single `.org` file.
- Every zettel has a UID generated by org's own `org-id-new`, so the format is
  controlled by `org-id-method`.
- Filenames follow `org-zk-filename-style`: by default `slug_uid.org` (a
  lowercased, underscore-separated title), or `Title (uid).org` when set to
  `title`.
- The same UID is stored in the file's `:ID:` property and registered with
  `org-id` at creation time, so native `id:` links resolve immediately.
- Tags live on a `#+filetags:` line in the `:tag1:tag2:` format.
- Links can be inserted bare (`[[id:UID]]`) or with the target's title as the
  description (`[[id:UID][Title]]`); each form has its own command, for both
  new and existing zettels.

## Installation

### Doom Emacs

In `packages.el`:

```elisp
(package! org-zk
  :recipe (:host github :repo "bucksbunni/org-zk"))
```

In `config.el`:

```elisp
(use-package! org-zk
  :commands (zk/new zk/find-file zk/search)
  :init
  (setq org-zk-directory "~/zk/")
  ;; Timestamp UIDs of the form YYMMDDHHMMSS, e.g. 260807150301
  (setq org-id-method 'ts
        org-id-ts-format "%y%m%d%H%M%S"))
```

Then `doom sync` and restart.

### Manual / straight.el

```elisp
(use-package org-zk
  :load-path "/path/to/zk"
  :config
  (setq org-zk-directory "~/zk/"))
```

## Configuration

| Variable           | Default              | Description                                  |
|--------------------|----------------------|----------------------------------------------|
| `org-zk-directory`     | `"~/zettelkasten/"`  | Where zettels live. `nil` = project root.    |
| `org-zk-filename-style` | `'slug`             | Filename template: `'slug` (`slug_uid.org`) or `'title` (`Title (uid).org`). |
| `org-id-method`    | (org default)        | Controls UID format (`'ts`, `'uuid`, etc.).  |
| `org-id-ts-format` | (org default)        | Timestamp format when `org-id-method` is `'ts`. |

## Commands

All commands are prefixed `zk/` and bound under `SPC n z` in Doom:

| Key         | Command                     | Description                              |
|-------------|-----------------------------|------------------------------------------|
| `SPC n z n` | `zk/new`                    | Create a new zettel and visit it         |
| `SPC n z r` | `zk/new-from-region`        | Extract the region into a new linked note |
| `SPC n z l` | `zk/new-with-link`          | Create a note and insert a bare link at point |
| `SPC n z L` | `zk/new-with-titled-link`   | Create a note and insert a titled link at point |
| `SPC n z i` | `zk/insert-link`            | Insert a bare link to an existing zettel |
| `SPC n z I` | `zk/insert-link-titled`     | Insert a titled link to an existing zettel |
| `SPC n z t` | `zk/add-tag`                | Add a tag to the current zettel          |
| `SPC n z T` | `zk/find-by-tag`            | Find a zettel by tag                     |
| `SPC n z f` | `zk/find-file`              | Fuzzy find by filename/title             |
| `SPC n z s` | `zk/search`                 | Fuzzy search across content (ripgrep)    |
| `SPC n z o` | `zk/follow-link-at-point`   | Follow the link at point                 |
| `SPC n z p` | `zk/preview-link-at-point`  | Preview the linked note in a side window |
| `SPC n z R` | `zk/rebuild-id-locations`   | Re-scan zettels and re-register IDs with org-id |
